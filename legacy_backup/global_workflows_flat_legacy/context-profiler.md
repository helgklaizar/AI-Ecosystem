# /context-profiler

## 📌 Описание
Скилл/Воркфлоу для обогащения контекста перед передачей его ИИ-агентам (Anti-RAG). Добавляет структурные метаданные к файлам кода или логам, чтобы LLM лучше понимала причинно-следственные связи.

## 🎯 Цели
1. Дать ИИ-агенту "бизнес-контекст" (уровень легаси, покрытие тестами, частота изменений).
2. Улучшить качество архитектурного ревью (`arch-evolution`) и баг-хантинга (`issue-triage`).

## 🚀 Шаги
1. **Target Selection**: Выбор файлов, попавших в выборку для агента (например, через поиск RAG).
2. **Metadata Extraction**:
   - Получение даты последнего коммита (`git log`).
   - Расчет цикломатической сложности.
   - Оценка процента покрытия тестами.
3. **Prompt Injection**:
   - Формирование блока "Context Profile" в начале промпта агента.
   - Пример: `[Context: Legacy Code, High Complexity, Changed 2 days ago]`.

## 🛠 Использование
Вызывается автоматически внутри агентов оркестраторов перед тем, как они отправят задачу специализированному кодеру (например, `huxley-coder`).
