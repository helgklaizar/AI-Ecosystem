# 🔐 Global Secrets Map

> **Инструкция для Агентов:** Этот файл описывает, ГДЕ лежат ключи и КАК устроена безопасность. **КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО** писать сюда сами значения токенов, пароли или секреты! Только пути!

## 🗺️ Топология секретов (Где что искать)

| Сервис / API | Среда (Prod/Dev) | Где физически лежит ключ | Переменная окружения (Env Var) |
| :--- | :--- | :--- | :--- |
| OpenAI / Anthropic / LLMs | Global | Системный уровень (`~/.bashrc` / `~/.zshrc`) | `OPENAI_API_KEY`, `ANTHROPIC_API_KEY` |
| Базы Данных (Postgres и т.д.) | Dev (Local) | Файл `.env.local` в корне проекта | `DATABASE_URL` |
| Базы Данных (Postgres) | Prod | Менеджер секретов сервера (Vercel/Docker Env) | `DATABASE_URL` |
| Внешние API (Telegram, Stripe) | Dev (Local) | Файл `.env.local` в корне проекта | `BOT_TOKEN`, `STRIPE_SECRET` |

## 🛡️ Аксиомы Безопасности для ИИ
1. **NO HARDCODE:** Никогда, ни при каких условиях не зашивать токены или пароли напрямую в код (`.ts`, `.py`, `.swift`).
2. **ENV LOAD:** При локальном запуске скриптов (Python/Node) всегда проверять загрузку через пакет `dotenv` или системный `os.environ`.
3. **GIT CHECK:** Перед коммитом всегда проверять, что файлы `.env`, `.env.local`, `*.pem`, `certs/` находятся в `.gitignore`.
