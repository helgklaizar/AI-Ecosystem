# Spec-Driven UI Pipeline

**Требования проекта:** Премиальные дизайны, Vanilla CSS (никакого Tailwind без запроса), HSL/динамические палитры, плавные анимации и Glassmorphism.

### Этап 1: Разбор спецификации
1. Пользователь передает описание UI или скриншот дизайна (используй возможность чтения изображений).
2. Выдели дизайн-токены: цвета (сгенерируй HSL палитру), типографику (Modern Google Fonts), тени, радиусы скругления.

### Этап 2: Фиксация дизайн-системы (ОБЯЗАТЕЛЬНО)
// turbo
3. Создай или обнови файл `DESIGN.md` в корне фронтенд-проекта. Файл **строго** должен быть в формате `@google/design.md`: 
   - YAML-блок (`---`) в начале с параметрами: `colors`, `typography`, `spacing`, `rounded`, `components`.
   - Markdown-блок с описанием (Do's, Don'ts) ниже.

### Этап 3: Foundation (Основа)
// turbo
4. Создай или обнови файловую структуру (например `src/index.css`), запиши утвержденные в `DESIGN.md` дизайн-токены как CSS-переменные в `:root`. 
5. Создай базовые утилитарные классы анимаций (hover-эффекты, fade-in, micro-interactions).

### Этап 4: Сборка компонентов
// turbo
6. Сгенерируй React / Vue / Vanilla компоненты, используя строго эти переменные. Избегай инлайн стилей (кроме сложных вычисляемых).
7. Реализуй динамику (интерактивы, состояния ошибок, hover'ы) чтобы интерфейс "ожил".

### Этап 5: Интеграция 
8. Интегрируй компоненты на целевую страницу приложения.
9. Проверь адаптивность и контрастность. Предложи пользователю запустить `build-local`/`npm run dev` для просмотра результата.
