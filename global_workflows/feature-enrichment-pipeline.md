# /feature-enrichment-pipeline

## 📌 Описание
Воркфлоу для автоматического обогащения "сырых" данных (текстов, постов, issue) явными признаками до их передачи в LLM. Реализует концепцию "Feature Engineering > Model Selection" для NLP-задач. Идеально подходит для конвейера Scout.

## 🎯 Цели
1. Отсеять информационный мусор ("Garbage in").
2. Вшить бизнес-логику в промпт для LLM в виде четких метрик.
3. Снизить затраты времени на инференс (за счет раннего отброса нерелевантных данных).

## 🚀 Шаги (Скиллы)
1. **Entity Extractor**:
   - Парсит входящий текст/JSON.
   - Извлекает сущности с помощью регулярок (Ссылки на GitHub/arXiv, упоминания MLX/Apple, тикеры).
2. **Hype Scorer**:
   - Вычисляет метрику значимости (например, ratio просмотров/репостов к среднему по каналу).
3. **Density Check**:
   - Рассчитывает "информационную плотность": отношение ключевых слов к общему объему текста.
   - Если плотность < порога, пост помечается как `skip`.

## 🛠 Интеграция
- Встраивается перед `mlx-news-tuner` или любым другим узлом суммаризации.
- Формирует расширенный JSON с объектом `features: {}` для передачи в системный промпт.
